# 隐私说明

本文说明当前代码能够处理哪些数据及其技术边界，供部署方制定面向用户的正式隐私告知。它不是法律意见，也不代表任何特定部署已经满足某项法律、认证或保留要求。

## 可能处理的数据

| 数据 | 例子 | 默认处理位置 |
| --- | --- | --- |
| 账号与安全 | 用户名、密码哈希、角色、状态、会话摘要、登录时间、IP、User-Agent | MySQL |
| 对话与反馈 | 用户消息、AI 回复、来源、点赞/点踩、会话时间 | MySQL |
| 知识库 | 简历/文档提取文本、网页内容、分块、向量、来源元数据 | ChromaDB |
| 职业数据 | 简历正文、岗位 JD、公司、投递阶段、截止日、面试问答/评分、报告、技能计划 | MySQL |
| 管理审计 | 管理动作、操作者、目标账号、时间和请求元数据 | MySQL |
| 运行数据 | 错误、健康、性能与限流状态 | 应用日志、Redis；取决于部署配置 |

项目默认不把上传原文件作为可下载资产长期保存，但会保存其提取文本和向量。部署方增加对象存储、日志平台、分析工具或备份后，实际数据位置会扩大，必须更新告知。

## 使用目的

数据用于提供账号安全、个人知识库、AI 求职问答、职业记录、统计、反馈改进、管理员运维和安全审计。部署方不得因为代码具备访问能力，就把用户数据用于未告知的招聘筛选、广告、画像训练或向第三方出售。

面试分数、岗位匹配和 AI 报告是辅助信息，不应自动用于对个人产生重大影响的决定。

## 外部处理与网络请求

- DeepSeek API 可能接收用户提示、相关聊天历史、系统提示和从知识库检索的片段。是否保存或用于改进服务，取决于部署方使用的供应商账号、区域和合同设置。
- 用户发起 URL 抓取时，目标网站会看到部署服务器的网络地址和常规 HTTP 请求信息；页面内容随后可能进入知识库或返回浏览器。
- 嵌入模型默认在后端运行；首次下载模型时，模型托管源会收到常规下载请求，不应收到用户文档正文。
- MySQL、Chroma、Redis、日志、备份与反向代理由实际部署方或其基础设施供应商处理。

部署方应在上线前列出真实供应商、处理地域、跨境情况、保留期与联系渠道；本仓库无法替代这些部署事实。

## 访问范围

- 普通用户 API 只能访问当前服务端会话归属的数据。
- 管理控制台可查看账号元数据、汇总指标、被反馈的 AI 回复片段和管理审计，但当前 API 不提供浏览用户全部职业数据或知识库正文的功能。
- 数据库、主机、备份和日志管理员可能在应用外接触数据，应采用最小权限、审批、审计和保密措施。

不要在简历或对话中提交与求职无关的身份证号、金融账号、健康信息、他人隐私或无权披露的机密资料。

## 保留、导出与删除

当前产品能力：

- `GET /api/career/export`：导出当前用户的简历、岗位、投递、面试、报告和技能数据。
- `DELETE /api/career/data`：明确确认后删除上述职业数据和当前用户 Chroma 知识库。
- 删除单个会话会删除该会话及消息；删除知识库来源会删除对应向量片段。
- 过期/撤销认证会话记录由配置的保留期和周期任务清理。

职业写入、导出、知识库写入与清空按用户获取 MySQL advisory lock，以避免协作请求在清空期间重新写入；这不是跨存储事务。清空先处理 Chroma、再事务删除 MySQL 职业表，失败后可安全重试，部署方仍应验证两侧结果。

导出会在后端内存中一次性组装该用户全部职业记录和面试题，不是分页或流式处理。数据量很大的账号可能遇到内存、延迟或超时限制；部署方应做容量评估，并提供失败后的替代导出流程。

重要边界：职业数据清空不删除账号、认证会话、聊天、反馈、管理审计、应用日志或已有备份；当前没有完整账号自助导出/删除 API。需要访问、更正或处置这些数据时，用户应联系实际部署方。部署方应定义可执行的验证、保留、法定例外与备份到期流程，而不是承诺即时从所有备份物理擦除。

## 备份与日志

MySQL 与 Chroma 应按同一批次备份。备份可能在在线数据删除后保留到既定轮换期限，因此必须加密、限制访问、记录恢复，并禁止作为日常查询副本。

应用日志应避免记录密码、Cookie、CSRF、API 密钥、完整提示词、简历或模型响应。调试模式可能增加输出，生产不应长期启用 `DEBUG`。

## 部署方上线清单

- 提供真实运营主体、隐私联系人和适用地区的用户告知。
- 说明必需/可选数据、处理目的、供应商、地域、保留和删除流程。
- 为未成年人、敏感数据、跨境、自动化决策和招聘场景完成适用评估。
- 配置最小权限、TLS、密钥管理、备份加密、日志脱敏和事件响应。
- 验证导出/删除结果以及 MySQL—Chroma 联动清理，并定期抽测恢复副本的到期销毁。
- 每次新增数据字段、外部 SDK、模型供应商或分析平台时同步更新告知。
